# 17.保持场景状态

## 📌 学习目标
- 保持场景状态

## 📖 内容笔记

### 1. 问题：敌人复活

杀死野猪 → 切换场景 → 再切换回来 → 野猪复活了。

**原因**：场景每次加载都是全新的实例，没有记录上一轮的状态。需要将场景状态保存在**独立于游戏场景的地方**——即 Autoload 的 `Game` 场景。

---

### 2. 场景状态数据结构

在 `game.gd` 中定义 `world_states` 字典，以场景名为 key、以状态字典为 value：

```gdscript
# game.gd
var world_states := {}    # { "forest": { "enemies_alive": [...] }, "cave": { ... } }
```

#### 数据示例

```
world_states = {
    "forest"  → { "enemies_alive" = ["Path/To/Boar1", "Path/To/Boar2"] },
    "cave"    → { "enemies_alive" = [] }
}
```

- Key：场景文件名（不含扩展名），如 `forest`、`cave`
- Value：该场景的状态数据，当前只记录 `enemies_alive`（存活敌人的**节点路径数组**）

---

### 3. Enemy 注册到 `enemies` 分组

方便统一遍历所有敌人，在 `enemy.gd` 的 `_ready()` 中注册：

```gdscript
# enemy.gd — _ready() 中新增
func _ready() -> void:
	add_to_group("enemies")
```

> 也可以直接在编辑器里手动加分组，效果相同。

---

### 4. 场景保存：`to_dict()`

切换场景前，把当前场景的状态序列化为字典。

```gdscript
# world.gd
func to_dict() -> Dictionary:
	var enemies_alive := []
	for node in get_tree().get_nodes_in_group("enemies"):
		var path := get_path_to(node)
		enemies_alive.append(path)

	return {
		enemies_alive = enemies_alive
	}
```

- 遍历 `enemies` 分组中的所有节点
- 用 `get_path_to(node)` 获取该节点在场景树中的路径
- 返回 `{ "enemies_alive": [路径数组] }`

---

### 5. 场景恢复：`from_dict()`

切换场景后，根据之前保存的数据恢复场景状态。

```gdscript
# world.gd
func from_dict(dict: Dictionary) -> void:
	for node in get_tree().get_nodes_in_group("enemies"):
		var path := get_path_to(node)
		if path not in dict.enemies_alive:
			node.queue_free()
```

- 遍历当前场景的所有敌人
- 如果某敌人的路径**不在** `enemies_alive` 数组中 → 说明已被杀死 → `queue_free()`

---

### 6. game.gd 完整 change_scene 流程

```gdscript
# game.gd — change_scene() 中新增逻辑

func change_scene(path: String, entry_point: String) -> void:
	var tree := get_tree()
	tree.paused = true

	var tween := create_tween()
	tween.set_pause_mode(Tween.TWEEN_PAUSE_PROCESS)
	tween.tween_property(color_rect, "color:a", 1, 0.2)
	await tween.finished

	# ── 新增：保存旧场景状态 ──
	var old_name := tree.current_scene.scene_file_path.get_file().get_basename()
	world_states[old_name] = tree.current_scene.to_dict()

	tree.change_scene_to_file(path)
	await tree.tree_changed

	# ── 新增：恢复新场景状态 ──
	var new_name := tree.current_scene.scene_file_path.get_file().get_basename()
	if new_name in world_states:
		tree.current_scene.from_dict(world_states[new_name])

	for node in tree.get_nodes_in_group("entry_points"):
		if node.name == entry_point:
			tree.current_scene.update_player(node.global_position, node.direction)
			break

	tree.paused = false

	tween = create_tween()
	tween.tween_property(color_rect, "color:a", 0, 0.2)
```

#### 场景名称提取

```gdscript
tree.current_scene.scene_file_path    # "res://world.tscn"
.get_file()                           # "world.tscn"
.get_basename()                       # "world"
```

#### 数据完整性保护

```gdscript
if new_name in world_states:          # 第一次进入该场景时 world_states 中没有对应 key
	tree.current_scene.from_dict(...)  # 跳过 from_dict → 避免字典访问报错
```

> [!tip] from_dict 的判断逻辑
> `if path not in dict.enemies_alive` — 用点号访问字典的 `enemies_alive` key，GDScript 中等价于 `dict["enemies_alive"]`。

### 7. 扩展性

`world_states` 字典不限于记录敌人生死，后续还可以扩展：

| 可存储的状态 | 字段名 | 数据类型 |
|------------|--------|---------|
| 宝箱是否已开启 | `chests_opened` | `Array[String]` |
| 敌人的剩余血量 | `enemy_health` | `Dictionary` |
| 已触发的剧情事件 | `events_triggered` | `Array[String]` |
