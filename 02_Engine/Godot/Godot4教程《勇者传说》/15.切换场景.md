# 15.切换场景

## 📌 学习目标
- 实现场景切换（Teleporter 传送门）
- 使用 Autoload 管理跨场景状态
- 重构 Player Direction 系统
- 将 Stats 迁移到全局自动加载场景

## 📖 内容笔记

### 1. Teleporter 脚本（传送门）

继承自上一期制作的 `Interactable`，覆盖 `interact()` 方法实现场景切换。

#### Teleporter.gd

```gdscript
class_name Teleporter
extends Interactable

@export_file("*.tscn") var path: String             # 目标场景路径（.tscn）
@export var entry_point: String = ""      # 目标入口点名

func interact() -> void:
	super()                               # 调用父类 interact（打印 + 信号）
	get_tree().change_scene_to_file(path)	# 立即销毁当前场景
```

> [!warning] 子类覆盖父类方法时，需手动调用 `super()`
> 子类定义了和父类同名的 `interact()`，父类的版本会被完全覆盖。如需执行父类逻辑，必须用 `super()` 显式调用。

---

### 2. EntryPoint 标记

用于指定玩家出生位置和方向的标记节点。

#### EntryPoint.gd

```gdscript
class_name EntryPoint
extends Marker2D

@export var direction: Player.Direction = Player.Direction.RIGHT

func _ready() -> void:
	add_to_group("entry_points")
```

>[!note] `Marker2D` 和 `Node2D` 功能几乎一样，但编辑器中十字准星更大，方便查找放置。

---

### 3. Game 自动加载场景（Autoload）

跨场景切换的核心：一个独立于普通游戏场景的自动加载场景，在整个游戏生命周期中始终存在。

#### game.tscn

- 根节点：`Node`，改名 `Game`
- 保存路径：`Globals/Game.tscn`

#### game.gd

```gdscript
extends Node

@onready var player_stats: Stats = $PlayerStats

func change_scene(path: String, entry_point: String) -> void:
	get_tree().change_scene_to_file(file_path)
	await get_tree().process_frame           # 等待一帧，新场景加载完成

	var nodes = get_tree().get_nodes_in_group("entry_points")
	for node in nodes:
		if node.name == entry_point_name:
			var current_scene = get_tree().current_scene
			if current_scene.has_method("update_player"):
				current_scene.update_player(node.global_position, node.direction)
			break
```

#### 项目设置 — 自动加载

项目设置 → 全局 → 自动加载 → 选择 `Globals/Game.tscn`（注意选 .tscn 不是 .gd）→ 添加

> 启用"全局变量"选项后，可在任意脚本中用 `Game` 访问该节点。

---

### 4. 场景世界脚本（update_player）

在场景的世界脚本（如 `World.gd`）中添加更新玩家位置的函数：

```gdscript
@onready var player: Player = $Player
@onready var camera: Camera2D = $Player/Camera2D

func update_player(position: Vector2, direction: Player.Direction) -> void:
	player.global_position = position
	player.direction = direction
	camera.reset_smoothing()    # 立即跳转到新位置，不做过渡动画
```

> `camera.reset_smoothing()`：切换场景后相机位置突然变化时，取消平滑过渡。

### 5. Player Direction 重构

原有代码直接操作 `graphics.scale.x`，需要重构为统一的 Direction 属性。
![[Pasted image 20260704141131.png]]
#### 重构步骤

1. 搜索所有 `direction` 用法 → 替换为 `movement`（代表水平移动方向 `-1/1`）
2. 新增 Direction 枚举和导出变量：

```gdscript
enum Direction {
	LEFT = -1,
	RIGHT = 1,
}

@export var direction: Direction = Direction.RIGHT:
	set(v):
		direction = v
		if not is_node_ready():          # 未 ready 时等待
			await ready
		graphics.scale.x = v
```

3. 将所有 `graphics.scale.x = ...` 改为 `direction = ...`
4. EntryPoint 的 `direction` 默认为 `Player.Direction.RIGHT`

> 遇到编辑器报错"Player 没有 direction" → 项目菜单 → 重新加载项目即可。

### 6. Stats 全局化

不同场景有各自的 Player 实例 → Stats 互相独立 → 切换场景后血量丢失。

#### 迁移方案
- 将 `Stats` 节点从 `Player` 场景**剪切**到 `Game` 场景，改名 `PlayerStats`
- `Player.gd` 中原 `@onready var stats: Stats = $Stats` → 改为 `Game.player_stats`
- `StatusPanel.gd` 中 `@export var stats: Stats` 重置（检查器选不到 Autoload 节点）→ 改为代码中默认使用 `Game.player_stats`：

```gdscript
@export var stats: Stats:
	set(v):
		stats = v
		if stats:
			stats.health_changed.connect(update_health)

func _ready() -> void:
	if not stats:
		stats = Game.player_stats
	update_health()
```

#### 首次同步跳过动画

切换场景后 `update_health()` 触发补间动画，导致血量闪烁。添加 `skip_animation` 参数：

```gdscript
func update_health(skip_animation: bool = false) -> void:
	var percentage := stats.health / float(stats.max_health)
	health_bar.value = percentage
	if skip_animation:
		delayed_health_bar.value = percentage
	else:
		create_tween().tween_property(delayed_health_bar, "value", percentage, 0.3)
```

> `_ready()` 中首次同步传 `true`，血量变化信号触发的同步不传（用默认 `false`）。

### 7. 场景搭建

#### MineGate 场景
- 根节点：`Teleporter`，保存于 `Objects/MineGate.tscn`
- 添加 `Sprite2D`：素材拖入 → `region` 勾选 → 像素吸附框选右下角门的图片
- ⚠️ 框选宽高确保**偶数**，否则相机移动时画面会飘
- 添加 `CollisionShape2D` → 矩形覆盖门范围

#### World 场景
- 移除上一期测试用的蘑菇/蔬菜节点
- 在 TileMap 下实例化 `MineGate`，摆放到右下角
- 复制一份 Gate，`Scale.x = -1` 翻转 → 放到平台上方
- 添加两个 EntryPoint：`TopEntry`（右上）、`BottomEntry`（右下）
- 两个门互相指向：上门的 entry_point = `BottomEntry`，下门的 entry_point = `TopEntry`
