# 21.结束界面

- 实现 Game Over 和 Game End 两种结束界面
- 理解 `_input()` vs `_unhandled_input()` 事件传播顺序
- 掌握打字机效果、Tween 回调链、echo 事件过滤

## 📖 内容笔记

### 1. Game Over Screen 场景

玩家死亡后显示暗色遮罩 + 文字。

```
Control (GameOverScreen)
├── ColorRect（全屏黑色，半透明，modulate fade in）
├── Label（"胜败乃兵家常事也，大侠请重新来过吧"）
│   └── 字体：32px，红色，深红阴影
└── AnimationPlayer（enter 动画）
```

> 保存于 `ui/game_over_screen.tscn`，在 Player 场景的 CanvasLayer 下实例化。

---

### 2. `_input()` vs `_unhandled_input()`

事件传播顺序：**`_input()` → 如果无人处理 → `_unhandled_input()`**

```gdscript
# game_over_screen.gd
func _input(event: InputEvent) -> void:
	get_window().set_input_as_handled()          # 标记已处理 → 阻止后续传播
	# ...
```

> `_input()` 优先于 `_unhandled_input()`，调用 `set_input_as_handled()` 后 ESC 不会触发 world.gd 的回城逻辑。

⚠️ **`_input()` 无法阻止 `Input.get_axis()` 的主动查询**——事件传播和输入状态查询是两个独立系统。不过死亡状态下 `move()` 根本不会被调用，所以不影响。

---

### 3. Echo 事件（回响事件）

长按键时操作系统会重复发送按下事件：

```
按 D 不放 → D (press) → D (echo) → D (echo) → ...
```

"按任意键继续"只需响应**第一次真正按下**：

```gdscript
if event.is_pressed() and not event.is_echo():    # 只响应 press，屏蔽 echo
```

### 4. game_over_screen.gd 完整代码

```gdscript
extends Control

@onready var animation_player: AnimationPlayer = $AnimationPlayer

func _ready() -> void:
	hide()
	set_process_input(false)            # 隐藏时不处理输入

func _input(event: InputEvent) -> void:
	get_window().set_input_as_handled()

	if animation_player.is_playing():    # 动画播完前不响应
		return

	if event is InputEventKey or event is InputEventJoypadButton or event is InputEventJoypadMotion:
		if event.is_pressed() and not event.is_echo():
			if Game.has_save():
				Game.load_game()
			else:
				Game.back_to_title()

func show_game_over() -> void:
	show()
	set_process_input(true)
	animation_player.play("enter")
```

### 5. Game Over 动画

AnimationPlayer → `enter` 动画（1.5s）：

| 时间 | 效果 |
|------|------|
| 0s → 0.6s | ColorRect `modulate.a`：0 → 1（缓入 easing） |
| 0.5s → 1.0s | Label `visible_ratio`：0 → 1（打字机效果） |

> **打字机效果**：调整 `Label > visible_ratio`（或 `visible_characters`），后者在出现第二行时不会把第一行往上推。

### 7. Game End Screen（多行字幕）

结构同 Game Over，但实现了一个**低配对话系统**——多行文字，每句之间有 fade 过渡。

```gdscript
# game_end_screen.gd
extends Control

const LINES := [
	"大魔王终于被打败了",
	"森林又恢复了往日的平静",
	"但这一切又值得嘛"
]

var current_line := -1
var tween: Tween

@onready var label: Label = $Label

func _ready() -> void:
	show_line(0)

func _input(event: InputEvent) -> void:
	if tween.is_running():
		return
	if event is InputEventKey or event is InputEventJoypadButton or event is InputEventJoypadMotion:
		if event.is_pressed() and not event.is_echo():
			if current_line + 1 < LINES.size():
				show_line(current_line + 1)
			else:
				Game.back_to_title()

func show_line(line: int) -> void:
	current_line = line

	tween = create_tween()
	tween.set_ease(Tween.EASE_IN_OUT).set_trans(Tween.TRANS_SINE)

	if line > 0:                                         # 非首句 → 先 fade out
		tween.tween_property(label, "modulate:a", 0, 1)
	else:                                                  # 首句 → 直接透明
		label.modulate.a = 0

	tween.tween_callback(label.set_text.bind(LINES[line])) # 换文字
	tween.tween_property(label, "modulate:a", 1, 1)        # fade in
```

> [!tip] `tween_callback` + `bind()` — 给 Tween 回调绑定参数
> `label.set_text.bind(LINES[line])` 等价于临时的匿名函数 `func(): label.set_text("xxx")`，Tween 会在时序到达时自动调用。

### 8. change_scene 的 duration 参数 + Bug 修复

#### 添加 duration 参数

```gdscript
# game.gd — change_scene() 开头
func change_scene(path: String, params := {}) -> void:
	var duration := params.get("duration", 0.2) as float
	# ...
	tween.tween_property(color_rect, "color:a", 1, duration)  # 替换写死的 0.2
```

`new_game()`、`back_to_title()`、`cave.gd` 中传入 `duration = 1` 让过渡更舒缓。

#### 🐛 Bug：快速连点出现画面闪烁

**原因**：第二个 fade in 的 Tween 忘记设置 `set_pause_mode` → 旧场景的 fade in 动画在新转场中被暂停 → 解除暂停后新旧两个 fade in 同时播放 → 黑屏提前消失。

**修复**：

```gdscript
# game.gd — 第二个 Tween 也加上 pause_mode
tween = create_tween()
tween.set_pause_mode(Tween.TWEEN_PAUSE_PROCESS)              # ← 修复
tween.tween_property(color_rect, "color:a", 0, duration)
```
