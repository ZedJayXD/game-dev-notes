# 攻击框和受击框

## 📌 学习目标
- 实现 Hitbox（攻击框） 和 Hurtbox（受击框）

## 📖 内容笔记

### 1. Hitbox 脚本（攻击框）

攻击框挂载在攻击方（玩家），用于检测是否命中目标。

#### hitbox.gd

```gdscript
class_name Hitbox
extends Area2D

signal hit(hurtbox)              # 命中敌人时发射

func _init() -> void:
	area_entered.connect(on_area_entered)

func on_area_entered(hurtbox: Hurtbox) -> void:
	hit.emit(hurtbox)            # 通知攻击方：我打中了目标
	hurtbox.hurt.emit(self)      # 通知受击方：你被打中了
```

> `_init()` 中连接 `area_entered`，注意 `Hitbox` 只检测 `Hurtbox`（通过碰撞层/掩码控制）。

---

### 2. Hurtbox 脚本（受击框）

受击框挂载在受击方（野猪），用于接收攻击信号。

#### hurtbox.gd

```gdscript
class_name Hurtbox
extends Area2D

signal hurt(hitbox)              # 被攻击时发射
```

> 简单来说：`Hitbox` 主动检测 → 命中 `Hurtbox` → 双向通知。

---

### 3. 场景节点搭建

| 节点 | 添加位置 | 子节点 | 说明 |
|------|---------|--------|------|
| `Hitbox` (Area2D) | Player 根节点下 | `CollisionShape2D` × N | N 段攻击对应 N 个攻击框 |
| `Hurtbox` (Area2D) | Enemy 根节点下 | `CollisionShape2D` × N | 身体各部位检测区域 |

- 给每个 `CollisionShape2D` 编辑形状（Rectangle / Circle）覆盖攻击范围
- 野猪的鼻子和尾巴也可以加 CollisionShape2D（但后续优化时会删掉）

---

### 4. 碰撞层设置

在"项目设置 → 图层 → 2D 物理"中：

| 层级 | 用途 |
|------|------|
| 第 3 层 | 攻击框（玩家 Hitbox → 检测敌人） |
| 第 4 层 | 受击框（敌人 Hurtbox → 接收检测） |

> 注意碰撞掩码：Hitbox 只检测 Layer 4，Hurtbox 只被 Layer 3 检测。

---

### 5. 动画轨道 K 帧（攻击框启用/禁用）

在 Player 的 `AnimationPlayer` 中，为每个攻击动画（attack1/2/3）的 `Hitbox.monitoring` 属性 K 帧：

| 动画阶段 | Hitbox.monitoring |
|---------|-------------------|
| 攻击前摇（举起武器） | ❌ 关 |
| 攻击判定帧（打到人） | ✅ 开 |
| 攻击后摇 | ❌ 关 |

> 这样只有在特定帧才会命中敌人，避免全程都有攻击判定。

---

### 6. Signal 信号连接

在 boar.gd（野猪脚本）中连接 Hurtbox 的信号：

```gdscript
func _on_hurtbox_hurt(hitbox: Hitbox) -> void:
	print("Poch!")  # 被打到时的回调
```

通过编辑器连接：选中 `Hurtbox` 节点 → 信号面板 → `hurt` → 连接到脚本。

---

### 7. monitoring vs monitorable（补充）

> 这两个属性在后续受伤/死亡章节中会用到，提前理解：

| 属性 | 所属节点 | 含义 |
|------|---------|------|
| `monitoring` | Hitbox | **能否去检测别人**。关闭后攻击框不再寻找目标（硬直时无法攻击） |
| `monitorable` | Hurtbox | **能否被别人检测到**。关闭后受击框不再被寻找（无敌帧/死亡后不受伤害） |
