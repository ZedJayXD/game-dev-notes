## 📖 内容笔记

### 1. 场景初始下落问题

场景加载后，玩家和野猪会从半空中短暂下落再落地。两个原因共同导致：

#### 原因 1：`is_on_floor()` 依赖于 `move_and_slide()`

场景刚开始还没有执行 `move_and_slide()`，所以 `is_on_floor()` 始终返回 `false`。

```gdscript
# player.gd 的 get_next_state 中有类似逻辑：
if not is_on_floor():
	return State.FALL    # 场景开始时会触发
```

#### 原因 2：TileMap 碰撞体延迟创建

TileMap 为了性能，不会在场景初始化时立即创建碰撞体，而是等待 1~3 帧（取决于帧率）。这期间玩家即便站在地上也会穿模。

#### 解决方案：转场遮丑

视频提到可以用 `StaticBody2D` 做临时落脚点，但太麻烦。更好的做法是用转场效果**遮住开头几帧的下落动画**。

### 2. 淡入淡出转场

在 Game 自动加载场景中添加一个全屏黑色 `ColorRect`，通过调整 Alpha 通道实现淡入淡出。

#### 场景结构

把 Game 根节点从 `Node` 改为 `CanvasLayer`：
- `layer = 99` — 确保在所有内容之上
- 子节点：`ColorRect`（铺满屏幕，黑色，Alpha=0）

#### 实际代码（game.gd）

```gdscript
extends Node

@onready var player_stats: Stats = $PlayerStats
@onready var color_rect: ColorRect = $ColorRect

func _ready() -> void:
	color_rect.color = 0    # 初始化：完全透明的黑色


func change_scene(path: String, entry_point: String) -> void:
	var tree := get_tree()
	tree.paused = true       # ① 暂停游戏

	var tween := create_tween()
	tween.set_pause_mode(Tween.TWEEN_PAUSE_PROCESS)    # ② 动画不受暂停影响
	tween.tween_property(color_rect, "color:a", 1, 0.2) # ③ 淡出（透明→黑）
	await tween.finished

	tree.change_scene_to_file(path)        # ④ 切换场景
	await tree.tree_changed                # ⑤ 等待新场景就绪

	for node in tree.get_nodes_in_group("entry_points"):
		if node.name == entry_point:
			tree.current_scene.update_player(node.global_position, node.direction)
			break

	tree.paused = false                    # ⑥ 恢复游戏

	tween = create_tween()
	tween.tween_property(color_rect, "color:a", 0, 0.2)  # ⑦ 淡入（黑→透明）
```

#### 流程时序图

```
pause → [0.2s fade out] → change_scene_to_file → await tree_changed
→ update_player → unpause → [0.2s fade in]
```

> [!tip] `await tree.tree_changed` vs `await process_frame`
> VTT 中使用 `await tree.process_frame`（Godot 4.1），实际项目代码使用 `await tree.tree_changed`（Godot 4.2+ 推荐方式）。

### 3. 转场期间暂停游戏

在转场过程中防止玩家移动、交互或受到伤害。

| 步骤 | 操作 |
|------|------|
| 转场开始 | `tree.paused = true` |
| 转场结束 | `tree.paused = false` |

#### 关键：让 Tween 在暂停时继续运行

```gdscript
tween.set_pause_mode(Tween.TWEEN_PAUSE_PROCESS)
```

否则 `tree.paused = true` 后连淡出动画本身也会卡住 → 游戏永远停在 `await tween.finished`。

### 4. Landing 状态 Bug 修复

切换场景后，玩家会进入 `LANDING` 状态做下蹲动作。

#### 原因

1. 场景初始时玩家从高空下落 → 触发 `FALL`，记录 `fall_from_y`
2. 随即 `update_player()` 把玩家瞬移到 EntryPoint
3. TileMap 碰撞体就绪后玩家落地，落差 > `LANDING_HEIGHT` → 触发 LANDING

#### 修复：同步更新 `fall_from_y`

在 `World.gd` 的 `update_player()` 函数中：

```gdscript
func update_player(pos: Vector2, dir: Player.Direction) -> void:
	player.global_position = pos
	player.direction = dir
	player.fall_from_y = pos.y    # 同步下落起点，防止误判 LANDING
	camera.reset_smoothing()
```

### 5. ColorRect 全屏设置

| 属性 | 值 |
|------|-----|
| 锚点预设 | 右下角"整个矩形" |
| `color` | `Color(0, 0, 0, 0)` — 黑色、完全透明 |
| CanvasLayer.layer | 99（最高层） |

## 💡 关键要点
1. **场景初始下落问题** [00:03](https://www.bilibili.com/video/BV1ZN4y1i7d4?t=3)
	- `is_on_floor()` 在首次 `move_and_slide()` 前始终为 `false` → 触发 FALL
	- TileMap 碰撞体延迟创建（1~3 帧）→ 玩家穿模
	- 方案：用转场效果遮住开头几帧，而非添加临时 StaticBody2D
2. **淡入淡出转场** [03:38](https://www.bilibili.com/video/BV1ZN4y1i7d4?t=218)
	- Game 根节点：`Node` → `CanvasLayer`，`layer = 99`
	- 添加 `ColorRect` 子节点：锚点→整个矩形，黑色、Alpha=0
	- `tween.tween_property(color_rect, "color:a", 1, 0.2)` 淡出
	- 切换场景 → `tween.tween_property(color_rect, "color:a", 0, 0.2)` 淡入
3. **转场期间暂停游戏** [08:55](https://www.bilibili.com/video/BV1ZN4y1i7d4?t=535)
	- `tree.paused = true` → 玩家不能移动/交互/受伤
	- ⚠️ 默认 Tween 也暂停 → 必须 `tween.set_pause_mode(Tween.TWEEN_PAUSE_PROCESS)`
	- 实际代码使用 `await tree.tree_changed`（Godot 4.2+ 推荐方式）
4. **Landing 误触修复** [06:36](https://www.bilibili.com/video/BV1ZN4y1i7d4?t=396)
	- 🔴 Bug：切换场景后 player 被瞬移，但 `fall_from_y` 仍是旧高度 → 落差超 `LANDING_HEIGHT` → 触发 LANDING
	- 🟢 Fix：`update_player()` 中同步 `player.fall_from_y = pos.y`
5. **game.gd 完整流程**：`pause → fade out(0.2s) → change_scene → tree_changed → update_player → unpause → fade in(0.2s)`
