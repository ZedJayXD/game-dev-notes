# 13.滑铲

## 📌 学习目标
- 实现人物滑铲

## 📖 内容笔记

### 1. 动画制作（三阶段）

滑铲分为三个阶段的动画：

| 动画名 | 帧数 | 总长 | 是否循环 | 功能 |
|-------|------|------|---------|------|
| `sliding_start` | 3 帧 | 0.15s（吸附 0.05） | ❌ | 躺下去 |
| `sliding_loop` | 3 帧 | 0.15s（吸附 0.05） | ✅ | 滑铲本体 |
| `sliding_end` | 2 帧 | 0.10s（吸附 0.05） | ❌ | 站起来 |

> [!tip] sliding_start 的吸附改为 0.05
> 滑铲动画如果用默认的 0.1s 吸附会感觉比较慢，调整为 0.05s 更干脆。

#### 滑铲作为闪避手段
- 在 `sliding_start` 动画的**开头**给 `Hurtbox` 节点的 `monitorable` 属性 K 帧 → **关闭**
  - `monitorable = false`：玩家的 Hurtbox 不会被敌人的 Hitbox 检测到 → **无敌**
- 在 `sliding_end` 动画中恢复 → `monitorable = true`（通过 reset 动画补全轨道）

### 2. 状态机新增状态

#### 状态枚举

```gdscript
enum State {
	# ... 已有状态 ...
	SLIDING_START,
	SLIDING_LOOP,
	SLIDING_END,
}
```

#### 新增变量

```gdscript
const SLIDING_DURATION := 0.3    # 滑铲持续时长（sliding_loop 的退出时间阈值）
const SLIDING_SPEED := 256.0     # 滑铲水平速度（略大于 run_speed）
const SLIDING_ENERGY := 4.0      # 单次滑铲消耗能量
const LANDING_HEIGHT := 100.0    # 触发 landing 的最小下落高度

@onready var slide_request_timer: Timer = $SlideRequestTimer

var fall_from_y: float           # 记录开始下落时的高度
```

#### get_next_state（状态转换逻辑）

```gdscript
func get_next_state(state: int) -> int:
	# 滑铲入场判断：是否应该滑铲
	if should_slide():
		return State.SLIDING_START

	match state:
		# ... 原有状态逻辑 ...
		
		State.SLIDING_START:
			if not animation_player.is_playing():
				return State.SLIDING_LOOP
		
		State.SLIDING_LOOP:
			# 触墙立即退出，或滑满时长后退出
			if is_on_wall() or state_machine.state_time > SLIDING_DURATION:
				return State.SLIDING_END
		
		State.SLIDING_END:
			if not animation_player.is_playing():
				return State.IDLE
```

> [!note] 关键修复
> `SLIDING_LOOP` 状态加入 `is_on_wall()` 检查，否则玩家会"面壁滑满 0.3 秒"——非常敬业。

#### transition_state（播放对应动画）

```gdscript
func transition_state(from: int, to: int) -> void:
	match to:
		State.SLIDING_START:
			animation_player.play("sliding_start")
			slide_request_timer.stop()              # 停止滑铲请求计时器
			stats.energy -= SLIDING_ENERGY          # 消耗能量
			fall_from_y = global_position.y         # 记录高度（用于 landing 判断）
		State.SLIDING_LOOP:
			animation_player.play("sliding_loop")
		State.SLIDING_END:
			animation_player.play("sliding_end")
		# ... 其他状态 ...
```

#### tick_physics（物理更新）

```gdscript
func tick_physics(state: int, delta: float) -> void:
	match state:
		State.SLIDING_START:
			slide(delta)
		State.SLIDING_LOOP:
			slide(delta)
		State.SLIDING_END:
			stand(default_gravity, delta)
```

- **SLIDING_START / SLIDING_LOOP**：调用 `slide(delta)`，朝固定方向滑行
- **SLIDING_END**：调用 `stand(delta)`，原地站起来

### 3. Slide 物理函数

```gdscript
func slide(delta: float) -> void:
	# 方向由玩家面朝方向决定
	var dir: float = -1.0 if graphics.scale.x < 0 else 1.0
	velocity.x = dir * SLIDING_SPEED
	velocity.y += default_gravity * delta
	move_and_slide()
```

- 水平速度固定（不受按键影响），方向由 `graphics.scale.x` 决定
- 垂直方向正常受重力影响

### 4. 滑铲条件判断（should_slide）

```gdscript
func should_slide() -> bool:
	# 1. 没有按下滑铲键（或计时器已过期）
	if not slide_request_timer.time_left > 0:
		return false
	# 2. 能量不足
	if stats.energy < SLIDING_ENERGY:
		return false
	# 3. 前方是否有空间（用 foot_checker 检测）
	if not floor_checker.is_colliding():
		return true     # 前方空旷，可滑铲
	# 前方有墙 → 换个方向或返回 false
	# 具体实现取决于 foot_checker 的配置
	return true  # 无 foot_checker 则直接返回 true，贴墙时滑铲不可用需手动处理
```

> [!warning] 为什么不能用 `is_on_wall()` 判断？
> `is_on_wall()` 只反映**上一次 `move_and_slide()`** 是否撞墙。如果玩家安安静静站在墙边，没有进行水平移动，`is_on_wall()` 始终返回 `false`。正确做法是用一个**脚部前方 RayCast2D** 主动检测。

### 5. 空中滑铲（Slide Request Timer）

类似土狼时间（Coyote Time）的做法，解决跳跃未落地时按键无效的问题。

#### 添加节点
- 复制 `JumpRequestTimer` 节点 → 重命名为 `SlideRequestTimer`
- 属性：`One Shot = true`，`Wait Time = 0.1`（与跳跃一致）

#### 信号连接（_input 事件）

```gdscript
func _input(event: InputEvent) -> void:
	if event.is_action_pressed("slide"):
		slide_request_timer.start()
	# ... 跳跃等逻辑 ...
```

#### 在 transition_state 中停止计时器

```gdscript
# 进入滑铲状态的任意阶段 → 停止计时器
if to in [State.SLIDING_START, State.SLIDING_LOOP, State.SLIDING_END]:
	slide_request_timer.stop()
```

### 6. Landing 优化（高度判断）

> 借鉴评论区"新墨"的思路：只有从比较高的地方掉下来才触发 landing，普通跳跃落地不触发。

#### stats.gd 新增变量

```gdscript
func _ready() -> void:
	# 将 health_changed / energy_changed 等信号连接 UI 更新函数
	stats.energy_changed.connect(update_energy)
	update_energy()
```

#### transition_state 中记录下落起点

```gdscript
func transition_state(from: int, to: int) -> void:
	match to:
		State.FALL:
			fall_from_y = global_position.y
```

#### get_next_state 中着陆判断

```gdscript
# 着陆：
if is_on_floor():
	var height = global_position.y - fall_from_y  # Y 向下为正
	if height >= LANDING_HEIGHT:
		return State.LANDING
	# ... 正常着陆逻辑 ...
```

### 7. 能量系统

#### stats.gd（新增能量相关）

```gdscript
signal energy_changed

@export var max_energy: float = 10.0
@export var energy_regen: float = 0.8     # 每秒恢复

@onready var energy: float = max_energy:
	set(v):
		v = clampf(v, 0, max_energy)
		if energy == v:
			return
		energy = v
		energy_changed.emit()

func _process(delta: float) -> void:
	energy += energy_regen * delta
```

> [!note] 为什么 energy 用 float 而 health 用 int？
> 能量需要随时间缓慢恢复（0.1、0.2 递增），必须用浮点数。

#### player.gd（能量消耗）

```gdscript
const SLIDING_ENERGY := 4.0  # 一次滑铲消耗 4 点能量

# should_slide() 中增加能量检查：
func should_slide() -> bool:
	if stats.energy < SLIDING_ENERGY:
		return false
	# ...

# transition_state 中实际扣减：
if to == State.SLIDING_START:
	stats.energy -= SLIDING_ENERGY
```

### 8. 能量 UI

#### 场景树调整
- `HealthBar` 节点 → 右键重置为子节点 → 用 `VBoxContainer`（改名 VB）包裹
- 新增 `TextureProgressBar`，改名 `EnergyBar`

#### TextureProgressBar 属性设置
| 属性 | 值 |
|------|-----|
| `under` → `Texture` | 细边框素材（自动裁剪）|
| `progress` → `Texture` | 细黄条素材（自动裁剪）|
| `max_value` | 1（归一化）|
| `step` | 0 |
| `value` | 1 |
| `progress_offset` | (1, 1) 纠正像素偏移 |
| `alignment` | Center（垂直居中）|

#### VBoxContainer 间距调整
- `theme_override_constants/separation` = 3（两个条之间的间距）

#### 更新函数

```gdscript
@onready var energy_bar: TextureProgressBar = $VB/EnergyBar

func update_energy() -> void:
	var percentage: float = stats.energy / stats.max_energy
	energy_bar.value = percentage

func _ready() -> void:
	stats.energy_changed.connect(update_energy)
	update_energy()
```
