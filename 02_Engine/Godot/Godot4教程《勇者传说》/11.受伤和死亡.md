# 11.受伤和死亡

## 📌 学习目标
- 实现受伤和死亡

## 📖 内容笔记

### 1. Stats 统计信息节点（血量系统）

新建 `Stats` 脚本（保存在 `classes/` 文件夹），管理角色的统计信息（血量等）。

#### stats.gd

```gdscript
class_name Stats
extends Node

@export var max_health: int = 3

@onready var health: int = max_health:
	set(v):
		v = clampi(v, 0, max_health)
		if health == v:
			return
		health = v
```

> [!info] export 与 @onready 的初始化顺序
> - **export 变量初始化**发生在普通变量初始化**之后**
> - 如果场景中把 `max_health` 改成 5，但 `health` 用普通变量 `var health = max_health`，拿到的仍是默认值 3
> - 加 `@onready` 后，初始化顺序变为：**先 export → 再 @onready**，问题解决
> - setter 中的 `clampi` 确保 `health` 永远在 `0 ~ max_health` 之间

在 Enemy 场景中添加 `Stats` 节点，在野猪场景中修改 `max_health`（如改为 5）。

### 2. Damage 伤害类

新建 `Damage` 脚本继承自 `RefCounted`（不是 Node），记录单次伤害信息。

#### damage.gd

```gdscript
class_name Damage
extends RefCounted

var amount: int        # 扣血量
var source: Node2D     # 伤害来源（Player / Enemy 的共同父类）
```

> [!note] 为什么用 RefCounted？
> - `RefCounted` 是 Godot 中最简单的、自带引用计数内存管理的类
> - 不需要手动 `free()`，引用归零自动释放
> - 比 `Node` 轻——不需要挂到场景树上
> - 再往下（`Object`）就需要手动内存管理，比较麻烦

### 3. 状态机修复：KEEP_CURRENT

> [!bug] 问题：连段攻击卡在 hurt 状态
> 当野猪处于 `HURT` 状态时再次受到伤害 → `pending_damage` 非空 → `get_next_state` 返回 `State.HURT` → 但返回当前状态值表示"**保持不变**"而非"**重新进入**" → 野猪卡死

#### state_machine.gd（修改）

```gdscript
const KEEP_CURRENT: int = -1

func _physics_process(delta: float) -> void:
	var next := get_next_state(state)
	if next == KEEP_CURRENT:
		return
	transition_state(state, next)
	state = next
```

- `get_next_state` 返回类型从 `State` 改为 `int`
- 返回 `KEEP_CURRENT`(-1) → 保持当前状态不变（不触发 transition）
- 返回有效状态值 → 重新进入该状态（触发 `transition_state`）
- 各脚本兜底的 `return state` 改为 `return StateMachine.KEEP_CURRENT`

### 4. 野猪受伤和死亡

#### 4.1 新增状态枚举

```gdscript
enum State {
	IDLE,
	WALK,
	RUN,
	HURT,    # 受伤
	DYING,   # 濒死
}
```

#### 4.2 变量定义

```gdscript
@onready var stats: Stats = $Stats

var pending_damage: Damage           # 待处理的伤害

const KNOCKBACK_AMOUNT := 512.0      # 击退力度
```

#### 4.3 受伤事件处理（\_on_hurtbox_hurt）

```gdscript
func _on_hurtbox_hurt(hitbox: Hitbox) -> void:
	pending_damage = Damage.new()
	pending_damage.amount = 1                 # 固定扣 1 血
	pending_damage.source = hitbox.owner      # 伤害来源 = 玩家
```

> [!tip] 设计思路：pending_damage 模式
> 不在信号处理函数里直接改状态，而是**记录待处理的伤害**，在 `get_next_state` 和 `transition_state` 中统一处理。状态转换逻辑集中在状态机中，更清晰。

#### 4.4 get_next_state（状态优先级）

```gdscript
func get_next_state(state: int) -> int:
	# 最优先：死亡
	if stats.health == 0:
		return State.DYING
	# 其次：受伤
	if pending_damage:
		return State.HURT

	match state:
		State.IDLE:
			if can_see_player():
				return State.RUN
			if state_machine.state_time > 2:
				return State.WALK
		State.WALK:
			if can_see_player():
				return State.RUN
			if wall_checker.is_colliding() or not floor_checker.is_colliding():
				return State.IDLE
		State.RUN:
			if not can_see_player():
				return State.WALK
		State.HURT:
			if not animation_player.is_playing():
				return State.RUN
		State.DYING:
			return StateMachine.KEEP_CURRENT

	return StateMachine.KEEP_CURRENT
```

#### 4.5 transition_state（扣血 + 击退 + 转向）

```gdscript
func transition_state(from: int, to: int) -> void:
	match to:
		State.IDLE:
			animation_player.play("idle")
			# ...（原有逻辑）
		State.WALK:
			animation_player.play("walk")
			# ...（原有逻辑）
		State.RUN:
			animation_player.play("run")
		State.HURT:
			animation_player.play("hurt")
			# 1. 扣血
			stats.health -= pending_damage.amount
			# 2. 击退：方向 = 从伤害来源指向自己
			var dir = pending_damage.source.global_position.direction_to(global_position)
			velocity = dir * KNOCKBACK_AMOUNT
			# 3. 被偷袭时转身面向玩家
			if dir.x > 0:
				direction = Direction.LEFT
			else:
				direction = Direction.RIGHT
			# 4. 清空待处理伤害
			pending_damage = null
		State.DYING:
			animation_player.play("die")
```

#### 4.6 die() 函数 + 动画方法调用轨道

```gdscript
func die() -> void:
	queue_free()
```

> [!tip] 动画末尾自动调用 die()
> 在 AnimationPlayer 的 `die` 动画中添加**方法调用轨道**（Method Call Track），在动画末尾插入关键帧调用 `die()`。死亡动画播完自动销毁节点，无需在 `get_next_state` 中判断动画是否结束。

#### 4.7 tick_physics（受伤/死亡时不动）

```gdscript
func tick_physics(state: int, delta: float) -> void:
	match state:
		State.IDLE:
			move(0.0, delta)
		State.WALK:
			move(max_speed / 3, delta)
		State.RUN:
			# ...（原有逻辑）
		State.HURT:
			move(0.0, delta)
		State.DYING:
			move(0.0, delta)
```

### 5. 动画制作

#### 5.1 die 动画（淡出效果）

- 复制 `hit` 动画 → 命名为 `die`
- 修改 `graphics` 节点的 `modulate` 属性：
  - 动画开头：白色（RGBA 全为 1）→ 关键帧
  - 动画末尾：Alpha 通道改为 0（完全透明）→ 关键帧
- 原理：`modulate` 颜色与节点内容**相乘**，Alpha=0 → 完全透明

#### 5.2 hurt/die 动画中的碰撞属性关键帧

| 属性 | 所在节点 | hurt 动画 | die 动画 | reset 动画 |
|------|---------|-----------|---------|-----------|
| `monitoring` | Hitbox | 关闭（硬直时不伤害玩家） | 关闭 | 勾选 |
| `monitorable` | Hurtbox | — | 关闭（死亡后不再受伤） | 勾选 |

> [!note] monitoring vs monitorable
> - `monitoring`：这个 Area2D **能否去检测别人**（关闭后 Hitbox 无法命中玩家）
> - `monitorable`：这个 Area2D **能否被别人检测到**（关闭后对方的 Hitbox 无法命中它）

#### 5.3 玩家 hurt/die 动画

- **hurt 动画**：4 帧，总长 0.4 秒，`h_frames = 4`
- **die 动画**：12 帧，总长 1.2 秒，`h_frames = 8`、`v_frames = 2`（首次用到两行素材）
- die 动画末尾同样添加方法调用轨道调用 `die()`
- 动画延长至 2.2 秒，让玩家"躺一会儿"再重新加载

### 6. 玩家受伤和死亡

逻辑与野猪基本一致，主要区别：

| 对比项 | 野猪 | 玩家 |
|-------|------|------|
| hurt 转向 | 被偷袭时转身面向玩家 | 不自动转向 |
| die() 函数 | `queue_free()` | `get_tree().reload_current_scene()` |
| die 动画时长 | 0.4s | 2.2s（含躺地效果） |

#### player.gd die() 函数

```gdscript
func die() -> void:
	get_tree().reload_current_scene()  # 重新加载当前场景
```

### 7. 攻击手感优化

#### 7.1 攻击前摇缩短

- `attack1` 和 `attack3` 动画前有举手前摇（0.3 秒），手感迟钝
- 将吸附改为 0.05，关键帧前移一格，总长度缩短（attack1: 0.5s，attack3: 0.7s）
- `attack2` 开头就是攻击动作，不需要改

#### 7.2 攻击框缩小

- 删除野猪多余的鼻子/尾巴 CollisionShape2D
- 缩小玩家 attack_shape 的范围（1/2/3 往回缩），减少"打到身后野猪"的情况
