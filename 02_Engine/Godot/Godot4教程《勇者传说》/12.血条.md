# 12.血条

## 📌 学习目标
- 添加血条

## 📖 内容笔记

### 1. 创建 StatusPanel 场景

新建场景，根节点类型选择 `HBoxContainer`（横向排列子控件的容器），改名 `StatusPanel`。

- 场景保存在 `UI/StatusPanel.tscn`
- 将 HBoxContainer 默认的 40×40 尺寸清零（让内容撑开）

#### 场景树预览
```
HBoxContainer (StatusPanel)
├── PanelContainer (Avatarbox)         ← 头像框背景
│   └── TextureRect (Avatar)           ← 头像
└── TextureProgressBar (HealthBar)     ← 血条
    └── TextureProgressBar (DelayedHealthBar)  ← 延迟血条（受伤动画用）
```

---

### 2. 头像框

#### 头像（TextureRect）

素材中没有专门的头像文件，使用**玩家动作素材**的头部区域。

- `TextureRect` 不像 `Sprite2D` 有 `region` 属性，只能用 `AtlasTexture`（图集纹理）
- 新建 `AtlasTexture` → 拖入玩家素材 → 切换到像素吸附 → 框选头部区域

#### 头像框（PanelContainer）

在 TextureRect 上右键 → 重设父节点 → 新建 `PanelContainer`。这是一种专门用来为控件提供背景的容器。

- 修改默认背景：`theme_override > styles > panel` → 新建 `StyleBoxTexture`
- 拖入头像框素材 → `Bkg > Edit Region` → 框选**右下角正方形框**（26×26 像素）
- 背景图片被缩小的问题：面板内容撑起来后，头像 11×11 但背景框 26×26
- Fix：`Layout > Custom Minimum Size` = `(26, 26)`

#### TextureRect Stretch Mode

`PanelContainer` 变大后会把 `TextureRect` 也拉大，`TextureRect` 默认会拉伸图片。

| Stretch Mode | 效果 |
|-------------|------|
| `Keep` | 1:1 原始尺寸 |
| `Keep Center` | 1:1 并居中 |
| `Keep Aspect Centered`（推荐） | 保持长宽比缩放并居中 |

选择 **`Keep Aspect Centered`**。

#### 内容边距（Content Margins）

设置 PanelContainer 与头像的间距：`theme_override > panel > StyleBoxTexture > Content Margins` → 全部改为 `3`

### 3. 血条搭建

使用 `TextureProgressBar`（基于纹理图片的进度条），它使用三张纹理叠在一起：

| 属性 | 说明 |
|------|------|
| `under` | 进度条背景（本视频未使用） |
| `progress` | 进度条本体（绿色） |
| `over` | 进度条边框 |

#### 设置步骤

1. **progress（绿条）**：新建 `AtlasTexture` → 拖入素材 → 自动裁剪 → 选择右边宽绿条
2. **over（边框）**：新建 `AtlasTexture` → 拖入素材 → 自动裁剪 → 选上面粗边框
3. **对齐修正**：绿条和边框之间有一像素偏移 → `progress_offset = (1, 1)`
4. **归一化**：`max_value = 1`，`step = 0`（禁用步长），`value = 1`（满血）
5. **垂直居中**：`Layout > Container Sizing > Vertical` → 改为 `Center`

### 4. 血条脚本（信号驱动）

#### status_panel.gd

```gdscript
extends HBoxContainer

@export var stats: Stats                # 接受外部传入的 Stats 节点
@onready var health_bar: TextureProgressBar = $HealthBar

func update_health() -> void:
	var percentage: float = stats.health / float(stats.max_health)
	health_bar.value = percentage

func _ready() -> void:
	stats.health_changed.connect(update_health)
	update_health()                         # 首次同步
```

> [!warning] 整数除法陷阱
> `stats.health` 和 `stats.max_health` 都是 `int`。GDScript 中 `int / int = int`，比如 2÷5=0（实际上应该是 0.4）。必须将除数或被除数转为 float：`float(stats.max_health)`。

#### stats.gd（添加信号）

```gdscript
signal health_changed

@onready var health: int = max_health:
	set(v):
		v = clampi(v, 0, max_health)
		if health == v:
			return
		health = v
		health_changed.emit()               # 血量变化 → 发射信号
```

### 5. CanvasLayer 屏幕空间 UI

在 Player 场景中实例化 `StatusPanel` 后：

- 在 `StatusPanel` 节点上右键 → 重设父节点 → 新建 `CanvasLayer`
- `CanvasLayer` 让 UI 脱离世界空间，始终在屏幕固定位置
- 调整位置到左上角

> 这样无论玩家在哪里、相机怎么移动，血条始终固定在屏幕左上角。

---

### 6. 受伤延迟效果（补间动画）

效果：受伤时绿色血条立即变短，红色血条**缓慢跟随**变短。

#### 实现方式

1. 选中 `HealthBar` → `Ctrl+D` 复制一份 → 拖成子节点 → 改名 `DelayedHealthBar`
2. **唯一化资源**：选 `DelayedHealthBar > Texture > Progress > AtlasTexture` → 右键 → 唯一化 → 把绿条改成红条
3. 清掉 `over`（边框）：因为它在绿条下面，边框不需要显示
4. **锚点**：工具栏锚点预设 → 右下角"整个矩形"
5. **显示顺序**：勾选 `Visibility > Show Behind Parent` → 红色始终在绿色下方
6. **当前进度测试**：把 `HealthBar.value` 改成 0.8 → 红色露出来

#### 补间动画代码

```gdscript
@onready var delayed_health_bar: TextureProgressBar = $HealthBar/DelayedHealthBar

func update_health() -> void:
	var percentage: float = stats.health / float(stats.max_health)
	health_bar.value = percentage
	# 红色血条延迟缓动
	create_tween().tween_property(delayed_health_bar, "value", percentage, 0.3)
```
