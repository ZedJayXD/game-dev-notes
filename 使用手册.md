# GameDev Vault 使用手册

> 本手册适用于独立游戏开发者，帮助管理学习笔记、项目文档和代码片段。

---

├── 02_Engine/             ← 引擎学习笔记

## 📝 笔记规范

### 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 学习笔记 | `主题名.md` | `CorgiEngine-斜坡移动原理.md` |
| 项目笔记 | `项目名-描述.md` | `造梦西游3-角色控制系统.md` |
| 代码片段 | `功能描述.cs.md` | `2D斜坡移动控制器.cs.md` |
| 设计文档 | `DESIGN-描述.md` | `DESIGN-战斗系统.md` |

### 附件管理规范

| 类型            | 存放位置                  | 命名格式             | 规则                            |
| ------------- | --------------------- | ---------------- | ----------------------------- |
| 截图            | `Assets/Screenshots/` | `描述-日期.png`      | **禁止** `Pasted image xxx.png` |
| Excalidraw 绘图 | `Assets/Excalidraw/`  | `主题名.excalidraw` | 嵌入用 `![[文件名.excalidraw]]`     |
| 其他图表          | `Assets/Diagrams/`    | `描述-日期.png`      | —                             |

### 学习笔记结构规范（教程系列用）

系列化教程笔记（如引擎教程、框架源码分析）按以下统一结构组织，每个 section 顺序固定：

```
## 📌 学习目标        ← 一句话说明本讲学什么
## 📖 内容笔记        ← 核心内容：概念原理 + 图文说明 + GDScript/C# 代码
## 💡 关键要点        ← 操作步骤（按顺序编号，快速回顾用）
## 🔗 参考来源        ← 视频链接、文档链接
## 📂 相关笔记        ← [[wikilink]] 系列导航 + Dataview
```

**Dataview 推荐写法**（用 frontmatter 的 `subject` 字段过滤，避免跨系列误匹配）：

```dataview
TABLE type, status, created
FROM ""
WHERE file.frontmatter.subject = this.subject AND file.name != this.file.name
SORT created ASC
```

**系列导航示例：**
```markdown
[[00.基础项目]] → [[01.相机]] → [[02.TileMap]] → [[03.视差背景]]
```

## ⚡ 推荐工作流

### 学习新技术时

1. 用 `T-学习笔记` 模板新建笔记
2. 边学边记，代码示例直接贴在笔记里
3. 学完后更新 `02_Engine/` 下的对应 MOC

### 开发项目时

1. 在 `01_Projects/` 下新建项目文件夹
2. 用 `T-项目笔记` 模板创建项目主页
3. 设计文档放在 `03_Design/` 下，项目笔记中用 `[[wikilink]]` 引用
4. 代码片段整理到 `04_CodeSnippets/`

### 定期维护

- 每周整理一次 `Assets/Screenshots/`，删除无用截图
- 每月更新一次 Dashboard 的「最近更新」列表
- 完成的项目移到 `07_Archive/`

## 📊 Dataview 查询速查

Dataview 是知识库的核心插件，用代码动态聚合笔记：

### 按标签列出笔记
```dataview
TABLE type, status, created
FROM #tag名
SORT created DESC
```

### 未完成任务看板
```dataview
TASK
FROM "01_Projects"
WHERE !completed
```

### 本月新建笔记
```dataview
TABLE file.ctime as "创建时间", type
FROM ""
WHERE file.ctime >= date(today) - dur(30 days)
SORT file.ctime DESC
```

## ❓ 常见问题

**Q：旧仓库的笔记怎么迁移？**
A：直接把 `.md` 文件复制到对应文件夹，用 Obsidian 的「检测潜在重命名」功能修复链接。

**Q：图片路径怎么管理？**
A：在 Obsidian 设置里把附件文件夹路径设为 `Assets/Screenshots/`，这样粘贴图片会自动放到正确位置。

**Q：怎么快速找到某个功能的笔记？**
A：用 `Ctrl+Shift+F` 全局搜索，或者用 Dataview 按 `tags` 筛选。

**Q：Templater 模板语法报错？**
A：确保用 Templater 的「Insert Template」命令插入模板，不要手动复制 `<% %>` 语法。
