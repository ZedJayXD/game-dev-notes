# GameDev Vault 使用手册

> 本手册适用于独立游戏开发者，帮助管理学习笔记、项目文档和代码片段。

---

## 📁 目录结构说明

```
GameDev Vault/
├── 00_Dashboard/          ← 仓库入口，总览和导航
├── 01_Projects/           ← 游戏项目文档（每个项目一个文件夹）
├── 02_Engine/             ← 引擎学习笔记
│   ├── Unity/             ← Unity 相关
│   │   └── CorgiEngine/   ← CorgiEngine 源码学习
│   └── Godot/             ← Godot 相关
├── 03_Design/             ← 游戏设计笔记
│   ├── Mechanics/         ← 机制设计
│   ├── LevelDesign/       ← 关卡设计
│   └── ArtStyle/          ← 美术风格参考
├── 04_CodeSnippets/       ← 可复用代码片段
├── 05_References/         ← 参考资料
│   ├── API/               ← API 文档摘录
│   ├── Tutorials/         ← 教程笔记
│   └── Books/             ← 书籍读书笔记
├── 06_Templates/          ← 笔记模板（勿直接编辑）
├── 07_Archive/            ← 已完成/废弃项目归档
├── 08_Daily/              ← 日报（每日开发记录）
├── 09_Issues/             ← 问题记录（Bug 追踪）
└── Assets/                ← 图片等资源文件
    ├── Screenshots/       ← 截图
    ├── Diagrams/          ← 图表/矢量图
    └── Excalidraw/        ← Excalidraw 绘图文件
```

## 📝 笔记规范

### 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 学习笔记 | `主题名.md` | `CorgiEngine-斜坡移动原理.md` |
| 项目笔记 | `项目名-描述.md` | `造梦西游3-角色控制系统.md` |
| 代码片段 | `功能描述.cs.md` | `2D斜坡移动控制器.cs.md` |
| 设计文档 | `DESIGN-描述.md` | `DESIGN-战斗系统.md` |

### 附件规范

- 截图统一放在 `Assets/Screenshots/` 下
- 附件命名格式：`描述-日期.png`，例如 `slope-physics-demo-20250618.png`
- **禁止**使用 `Pasted image xxx.png` 这样的默认名

## 🔖 模板使用

Templater 已配置**文件夹模板**，在以下文件夹新建文件时自动套用对应模板：

| 文件夹 | 自动套用模板 | 用途 |
|--------|-------------|------|
| `01_Projects/` | `T-项目笔记.md` | 项目主页 |
| `02_Engine/` | `T-学习笔记.md` | 引擎学习笔记 |
| `04_CodeSnippets/` | `T-代码片段.md` | 代码片段 |

手动使用模板（命令面板 → `Templater: Insert Template`）：

| 模板 | 用途 |
|------|------|
| `T-学习笔记.md` | 记录新技术/框架的学习过程 |
| `T-项目笔记.md` | 管理游戏项目的开发进度 |
| `T-代码片段.md` | 保存可复用的代码 |
| `T-日报.md` | 每日开发记录（今日完成/遇到问题/明日计划） |

| `T-Excalidraw.md` | Excalidraw 绘图笔记 |

## ⚡ 推荐工作流

### 学习新技术时

1. 用 `T-学习笔记` 模板新建笔记
2. 边学边记，代码示例直接贴在笔记里
3. 学完后更新 `02_Engine/` 下的对应 MOC

### 开发项目时

1. 在 `01_Projects/` 下新建项目文件夹
2. 用 `T-项目笔记` 模板创建项目主页
3. 设计文档放在 `03_Design/` 下，项目笔记中用 `[[wikilink]]` 引用
4. 代码片段整理到 `04_CodeSnippets/`

### 定期维护

- 每周整理一次 `Assets/Screenshots/`，删除无用截图
- 每月更新一次 Dashboard 的「最近更新」列表
- 完成的项目移到 `07_Archive/`

## 📊 Dataview 使用技巧

Dataview 是知识库的核心插件，用代码动态聚合笔记：

### 常用查询模板

**按标签列出笔记：**
~~~dataview
TABLE type, status, created
FROM #tag名
SORT created DESC
~~~

**未完成任务看板：**
~~~dataview
TASK
FROM "01_Projects"
WHERE !completed
~~~

**本月新建笔记：**
~~~dataview
TABLE file.ctime as "创建时间", type
FROM ""
WHERE file.ctime >= date(today) - dur(30 days)
SORT file.ctime DESC
~~~

## ❓ 常见问题

**Q：旧仓库的笔记怎么迁移？**
A：直接把 `.md` 文件复制到对应文件夹，用 Obsidian 的「检测潜在重命名」功能修复链接。

**Q：图片路径怎么管理？**
A：在 Obsidian 设置里把附件文件夹路径设为 `Assets/Screenshots/`，这样粘贴图片会自动放到正确位置。

**Q：怎么快速找到某个功能的笔记？**
A：用 `Ctrl+Shift+F` 全局搜索，或者用 Dataview 按 `tags` 筛选。

**Q：Templater 模板语法报错？**
A：确保用 Templater 的「Insert Template」命令插入模板，不要手动复制 `<% %>` 语法。
