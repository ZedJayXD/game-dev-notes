## Obsidian MCP 自签证书连接修复

WorkBuddy 尝试连接 obsidian MCP 时报错：`self signed certificate`（Node.js fetch 拒绝自签 TLS 证书）。

### 根因
- Obsidian Local REST API 插件 HTTPS 端口（27124）使用自签证书
- WorkBuddy 的 MCP http client 基于 Node.js fetch，默认不信任自签证书
- 插件虽有 HTTP 端口（27123），但 `enableInsecureServer` 默认 false 未开放

### 修复方案（用户选定：改用 HTTP 端口）
1. `data.json`：`enableInsecureServer: false` → `true`（开放 HTTP 27123）
2. `mcp.json`：`url` 从 `https://127.0.0.1:27124/mcp/` → `http://127.0.0.1:27123/mcp/`
3. 本地回环不走 TLS，规避自签证书问题，无安全风险

### 待用户操作
- 在 Obsidian 设置 → 社区插件 → 关闭再开启 Local REST API（或重启 Obsidian），使 HTTP 27123 端口开放
- 完成后在 WorkBuddy 连接器管理点「信任/启用」obsidian

### 备选方案（未采用）
- 导出插件自签证书（在 `data.json` 的 `crypto.cert` 字段）为 .pem，设 `NODE_EXTRA_CA_CERTS` 环境变量，重启 WorkBuddy —— 保持 HTTPS 但需配系统环境变量
